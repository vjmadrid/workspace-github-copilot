# Enfoque de aproximación de ficheros de instrucciones

Este ejercicios ayudará a descubrir las habilidades de cada persona dentro de un fichero de instrucciones de Copilot

## Objetivo

Crear ficheros de instrucciones que adapten el comportamiento de Copilot a tu estilo, tu rol y el tipo de proyecto

## Tareas

* Introducción conceptual (10 min)
* Exploración inicial (15 min)
  * Identificar los parámetros que definen tu entorno de desarrollo: lenguajes principales, frameworks usados, estilos de código preferidos (camelCase, PascalCase, uso de comentarios, docstring, etc.)
  * Nivel de detalle de las respuestas
  * Analizar el propósito de las instrucciones personalizadas: diferencias entre “qué quiero que sepa de mí” y “cómo quiero que responda”
  * Ejemplo de estructura de fichero y zonas clave
* Diseño guiado (30 min)
  * Crear el primer fichero con instrucciones sobre:
    * Estilo de código (lenguaje, formato, comentarios)
    * Nivel de detalle en explicaciones.
    * Contexto del proyecto (frameworks, arquitectura, normas internas)
* Práctica de validación (15 min)
  * Probar las instrucciones en un entorno de desarrollo
  * Evaluar los resultados: ¿responde como esperabas?, ¿qué ajustes harías?
* Iteración y optimización (15 min)
  * Ajustar secciones del fichero hasta que las respuestas sean coherentes
  * Agregar notas de comportamiento esperado
* Actividad de reflexión (10 min)
  * Documentar lo aprendido y crear una plantilla reutilizable de instrucciones personalizadas
